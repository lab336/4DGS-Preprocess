# 相机矩阵排列工具

脚本：`preprocess/camera_grid_layout_tool.py`

用途：让客户只用一张代表图确认一次固定相机阵列的二维物理排列，并把该排列复用于全部拍摄帧。一个物理位置可以放置一台、两台或更多相机，适用于机位很近但焦距、曝光或用途不同的相机组。工具只保存原图路径和相机 ID，不复制、重命名或修改图片。

## 启动

```powershell
conda run --no-capture-output -n gsstatic python preprocess/camera_grid_layout_tool.py
```

也可以指定初始矩阵大小或继续编辑已有项目：

```powershell
python preprocess/camera_grid_layout_tool.py --rows 10 --cols 10
python preprocess/camera_grid_layout_tool.py --layout output/camera_grid.json
```

依赖 Tkinter；安装 Pillow 后显示 JPG/TIFF 等缩略图。`gsstatic` 环境已经包含二者。没有 Pillow 时仍可按文件名排列，但不显示缩略图。

## 推荐操作流程

1. 输入矩阵行数和默认列数，点击“创建/重设矩阵”。
2. 加载代表图片：
   - “加载图片”：选择同一帧中每台相机的一张图片，相机 ID 默认是文件名。
   - “加载相机目录”：选择包含多个相机子目录的根目录；每个子目录视为一台相机，并自动取自然排序后的第一张图片作为预览，相机 ID 是目录名。
3. 可以先用“自然逐行”或“蛇形排列”自动填充，再进行少量人工调整。
4. 从左侧选择相机后点击空位置，或者直接拖入位置卡片：
   - 拖入空位置：建立一个新相机位置；
   - 拖入已经有相机的位置：追加为同位置相机组，不覆盖原相机；
   - 拖动已放置相机：把该相机从原位置移动到目标位置，组内其他相机不受影响；
   - 拖动过程中会显示相机缩略图浮层、目标高亮和松开后的结果提示。
5. 每行左侧的 `− / +` 独立调整该行位置数。右键任意位置还可以：
   - 把左侧当前选中的相机加入该位置；
   - 选择或移除同位置组内的某一台相机；
   - 清空该位置的全部相机；
   - 标记/恢复物理空洞；
   - 在任意位置前后插入或删除位置。
6. 选择四邻域或八邻域及半径，点击“检查布局”，确认没有孤立相机或不连通区域。
7. 点击“导出布局”。

工具支持 `Ctrl+Z` 撤销、`Ctrl+Y` 重做、`Ctrl+S` 保存、`Ctrl+O` 打开布局。格子大小滑块和双向滚动条用于编辑大型相机矩阵。

## 邻接关系

- **四邻域**：只连接同一行或同一列、指定半径内的上下左右相机，不会把对角线算进去。
- **八邻域、半径 1**：再加入四个对角邻居，通常更适合二维摄影棚阵列。
- **半径 2 或更大**：允许跨越更多格子，仅在相邻相机重叠仍不足时使用。
- **同位置相机组**：组内相机两两生成邻接关系；该组中的每台相机都会与周边位置的每台相机建立邻接关系。例如一个位置有广角和长焦、相邻位置有一台相机，会生成“广角—长焦”“广角—邻居”“长焦—邻居”三对。
- 标记为“物理空洞”的格子不会产生邻接关系；普通未放置空格仍保留为可编辑位置。

## 导出文件

假设选择 `camera_grid.json`，工具会生成四个文件（每个文件均采用临时文件替换，避免写到一半留下残缺文件）：

| 文件 | 用途 |
| --- | --- |
| `camera_grid.json` | 完整、可重新打开编辑的规范数据；每个位置通过有序 `camera_ids` 数组保存一台或多台相机。 |
| `camera_grid_matrix.txt` | 便于人工阅读/审核的制表符矩阵；同位置相机使用 `相机A \| 相机B` 表示，`_` 是可用空位置，`-` 是物理空洞。 |
| `camera_grid_matrix.csv` | 一台相机一行；同行同列表示同一物理位置，`slot_order` 和 `slot_camera_count` 表示组内顺序及数量。空位置和物理空洞仍各保留一行。 |
| `camera_grid_neighbour_pairs.txt` | 按当前四/八邻域和半径生成的相机邻接对白名单。 |

直接加载单帧图片时，相机 ID 是完整图片文件名，因此在 ID 不含空格的情况下，邻接文件可直接用于：

```powershell
python colmap/superglue_colmap.py `
  --pair_mode pairs_file `
  --pairs_file camera_grid_neighbour_pairs.txt
```

使用“相机目录”模式时，邻接文件保存的是相机目录名。规范 JSON 同时保存了目录与预览图路径，后续标定流程应通过相机目录 ID 或 `camera_mapping.json` 做映射。

布局格式当前为版本 2。工具可以直接打开版本 1 的单相机位置布局，并在下次保存时自动升级；原位置、物理空洞和相机 ID 都会保留。

## 数据安全与检查

- 删除位置、清空布局或删除项目中的相机都不会删除原始图片。
- 缩短某一行或设置物理空洞时，其中的一台或多台相机会全部回到左侧“未放置”列表。
- 导出前若存在未放置、孤立或不连通相机，工具会给出提示，但允许导出未完成草稿。
- JSON 是主文件；TXT/CSV/邻接文件都可以从 JSON 重新生成。
